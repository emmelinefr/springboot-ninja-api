# 🥷 Ninja Management API

Backend REST API developed with **Java + Spring Boot**, focused on applying production-oriented backend practices such as layered architecture, database migrations, and RESTful design.

---

## 🎯 Purpose

This project was built as a practical exercise to strengthen core backend engineering skills, including:

- REST API design
- Layered architecture
- Relational data modeling
- Database versioning
- Clean code practices

---

## 🧰 Tech Stack

- Java 17+
- Spring Boot
- Spring Data JPA / Hibernate
- H2 Database (development)
- Flyway (database migrations)
- Swagger / OpenAPI
- Maven
- Git / GitHub

---

## 🏗️ Architecture

The project follows a **layered architecture pattern**:

Controller → Service → Repository

Key design decisions:
- Separation of concerns
- DTO usage for API contracts
- Business logic isolated in service layer
- Clean and maintainable structure

---

## 🔗 Data Model

Implemented relational mappings using JPA:

- One-to-Many relationships
- Many-to-One relationships

Focused on consistency and proper entity design.

---

## 🔄 API Design

- RESTful principles applied
- Full CRUD operations
- Partial updates using PATCH
- Proper HTTP method usage
- DTO-based request/response structure

---

## 🧬 Database

- H2 in-memory database for development
- Flyway migrations for schema versioning
- Reproducible database state across environments

---

## 📄 API Documentation

- Swagger / OpenAPI integrated
- Interactive endpoint testing
- Clear request/response contracts
